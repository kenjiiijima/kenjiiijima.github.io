# GitHub Pages でホームページを作る  
――これだけ見れば、実際に立ち上がる

この文章は、  
GitHub Pages を使って、実際にホームページを立ち上げるための作業記録である。

このページを上から順に読み、  
同じ操作を行えば、最低限の形でホームページが立ち上がる。

---

### 用語について

このページでは、  
GitHub Pages を使ってホームページを立ち上げるための  
最低限の用語を用いている。

途中で意味が分からなくなった場合は、  
次の用語ページを参照されたい。

→ [用語の説明（GitHub Pages）](glossary-github.md)

---

## Step 1　GitHub にサインアップする

まず、GitHub のアカウントを作る。

1. ブラウザで https://github.com にアクセスする  
2. 画面右上の「Sign up」を選ぶ  
3. ユーザー名を決める  

ユーザー名は、  
あとで作るホームページの URL に使われる。

この時点で完璧である必要はない。

---

## Step 2　Pages 用リポジトリを作る

ログイン後、ホームページ本体を置く場所を作る。

1. 画面右上の「＋」を押し、「New repository」を選ぶ  
2. Repository name に、次の形式で名前を入力する  

　　ユーザー名.github.io  
　　例えば、tarou.github.io  

ここで入力するのは、  
Step 1 で決めた自分のユーザー名の後ろに、  
「.github.io」をそのまま付けた文字列である。

3. 設定は、次の2点だけ確認する  

　　Public  
　　Add README.md を ON  

4. 「Create repository」を押す  

これで、ホームページの土台ができる。

---

## Step 3　README.md を編集する（トップページ）

README.md は、そのままホームページのトップページになる。

1. README.md を開く  
2. 鉛筆マーク（Edit）を押す  

ここで、最初の文章を書いてみる。

　　太郎のホームページ  
　　わたくし太郎は、ここに研究記録を書いてゆきます。

この2行は、  
「何を書けばよいか分からない」場合の一例である。  
内容は、あとから何度でも書き換えられる。

3. 「Commit changes」を押す  

---

## Step 4　表示を確認する

ブラウザで、次の URL を開く。

　　https://ユーザー名.github.io  

ここで表示される内容は、  
Step 3 で書いた README.md の文章である。

反映まで、少し時間がかかることがある。

表示されない場合は、  
Ctrl + F5（強制更新）を試す。

---

## Step 5　タイトルが自動表示される場合（必要なときだけ）

ページの一番上に、  
意図していないタイトルが表示されることがある。

その場合は、設定ファイルを1つ追加する。

1. 「Add file」→「Create new file」を選ぶ  
2. ファイル名に、次の文字列を入力する  

　　_config.yml  

3. ファイルの中に、次の2行を書く  

　　title:  
　　description:  

何も書かないこと自体が設定になる。

4. 「Commit changes」を押す  

これで、余計な表示は消える。

---

## Step 6　よく止まりやすいところ

表示されない場合、  
多くは失敗ではなく「反映待ち」である。

URLを知らない人には、基本的に見られない。  
この段階で、公開・非公開を気にしすぎる必要はない。

---

## Step 7　ここまでで出来ていること

この時点で、すでに次の状態になっている。

・無料  
・広告なし  
・自分専用の URL  
・あとから修正できる  
・途中のまま置いておける  

これで、ホームページは立ち上がっている。
