# 用語（GitHub Pages）

このページは、GitHub Pages を使ってホームページを作る途中で出てくる言葉を、
**不安になったときに確認するため**にまとめたものです。

覚える必要はありません。「そういう意味か」と分かれば十分です。

---

## GitHub

文章やファイルを置いて、変更の履歴を残しながら管理できる場所です。
上書きしても過去に戻れます。

---

## リポジトリ（repository）

GitHub の中にある「一つの置き場」です。
今回の場合は、**このリポジトリ＝このホームページ**だと思えば十分です。

---

## GitHub Pages

GitHub に置いたファイルを、そのままホームページとして公開できる仕組みです。
サーバー契約は不要で、広告も出ません。

---

## .github.io

GitHub が管理している公開用ドメイン（URL）です。
`ユーザー名.github.io` という形で自分のサイトになります。

---

## .io とは

`.io` はもともとインド洋地域（British Indian Ocean Territory）の国別ドメインです。
現在は技術系で使われることが多く、input/output の連想もあって広く定着しています。
深く気にする必要はありません。

---

## README.md

この置き場に来た人が最初に読む文章です。
GitHub Pages では README.md の内容がトップページとして表示されます。

---

## .md（Markdown）

Markdown 形式のファイルです。
普通の文章に近い形で、見出しやリンクなどを書けます。

---

## Commit（コミット）

変更を「記録として確定する」操作です。
保存して履歴に残す、の合図だと思ってください。

---

## _config.yml

GitHub Pages（Jekyll）の設定ファイルです。
サイト名や表示の細かい調整などを書きます。空でも問題ありません。
