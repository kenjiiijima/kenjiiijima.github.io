# 用語の説明（GitHub Pages）

このページでは、  
GitHub Pages を使ってホームページを立ち上げる際に出てくる  
主な用語を簡潔に説明する。

---

## GitHub（ギットハブ）

ソースコードや文章、設定ファイルなどを  
保存・管理するためのオンラインサービスである。

---

## リポジトリ（repository）

GitHub 上に作る、ファイル一式をまとめた単位である。  
変更の履歴が自動的に記録される。

---

## GitHub Pages

GitHub に置いた内容を、  
そのままホームページとして公開できる仕組みである。

---

## README.md

リポジトリ内に置く説明用のファイルである。  
GitHub Pages では、この内容がトップページとして表示される。

---

## .github.io

GitHub Pages 用に割り当てられる  
ホームページ専用のドメイン名である。

例：tarou.github.io

---

## _config.yml

GitHub Pages の表示を調整するための設定ファイルである。  
ページタイトルなどの表示制御に用いられる。

---

## Commit（コミット）

ファイルの変更内容を確定し、  
GitHub に保存する操作である。

---

## Public

リポジトリの公開設定である。  
誰でも内容を閲覧できる状態を指す。

---

## Private

リポジトリの非公開設定である。  
指定した人のみが内容を閲覧できる状態を指す。
